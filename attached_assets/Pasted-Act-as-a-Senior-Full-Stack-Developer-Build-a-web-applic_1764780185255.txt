### 1. Dependencies & Configuration
- Use flask-login for authentication.
- Use google-generativeai for summarizing medical history.
- Use fpdf for generating custom PDF prescriptions.
- Use werkzeug.security for password hashing.
- Load GEMINI_API_KEY from environment variables.

### 2. Database Models (Relational)
Create these models with appropriate relationships:
1. *User:* id (PK), unique_id (Custom), role (Patient, Doctor, Pharmacist, Lab, Receptionist), name, phone, age, gender, password_hash.
   - Doctor Specifics: specialization, hospital_name, medical_license_id, address, website, email.
   - Patient Specifics: raw_medical_history (text), ai_summary_history (text).
2. *MedicineInventory:* id, name, stock_quantity (Managed by Pharmacist).
3. *Appointment:* link Patient & Doctor. status (Waiting, In-Consultation, Completed).
4. *Prescription:* link Patient & Doctor. content (JSON: meds, dosage), diagnosis, height, vitals, pdf_path.
5. *LabRequest:* link Patient, Doctor, & LabUser. status (Pending, Done).

### 3. Logic & ID Generation
- *Patient ID Gen:* On registration, auto-generate unique_id = Last 4 digits of Phone + First 3 chars of Name.
- *Gemini AI Integration:* During Patient Registration, take the raw_medical_history input, send it to Gemini API with the prompt "Summarize this medical history into 3 bullet points", and save the result to ai_summary_history.
